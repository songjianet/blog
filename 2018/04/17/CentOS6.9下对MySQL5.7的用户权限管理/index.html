CentOS6.9下对MySQL5.7的用户权限管理

发表于 2018-04-17 | 分类于 Linux | 阅读次数:
字数统计: 629 字 | 阅读时长 ≈ 2 分钟

在实际开发中的很多情况下，我们不免要对数据库根据用户的不同进行权限的分配。在MySQL数据库中，基本有28种权限，我们可以将其归类成三大类的权限。

MySQL的权限分类

一）数据操作类
操作含义
SELECT查询权限
INSERT插入权限
UPDATE更新权限
DELETE删除数据权限
FILE文件访问权限
二）结构操作类
操作含义
CREATE创建数据库、表或索引权限
ALTER更改表权限（添加字段或者索引等）
INDEX索引权限
DROP删除数据库或表权限
CREATE TEMPORARY TABLES创建临时表权限
SHOW VIEW查看视图权限
CREATE ROUTINE创建存储过程权限
ALTER ROUTINE更改存储过程权限
EXECUTE执行存储过程权限
CREATE VIEW创建视图权限
EVENT可在数据下创建时间调度器权限
TRIGGER可在数据下创建触发器权限
三）管理操作类
操作含义
GRANT赋予权限选项
SUPER执行kill线程权限
PROCESS查看进程权限
RELOAD执行flush-hosts，flush-logs等权限
SHUTDOWN创建临时表权限
SHOW DATABASES关闭数据库权限
LOCK TABLES可对其下所有表进行锁定权限
REFERENCES未来MySQL特性的占位符
REPLICATION CLIENT复制权限
REPLICATION SLAVE复制权限
CREATE USER创建用户权限
